## 📌 数据采集核心原则：宁滥勿缺 (已达成)
本项目作为“低空经济大数据决策沙盘”的核心数据底座，在前期数据采集阶段贯彻“宁滥勿缺”原则。目前的弹药库已极其充足，包含了南山区建筑基座、医院学校避让点、高精度无人机测试轨迹集。

## 📁 目录结构管理规范
*   **`data/`** (数据根目录)
    *   **`raw/`**: **【封存】原始数据池**。按城市划分（如 `shenzhen/`, `beijing/` 等）存放从 Hugging Face 和 Overpass API 抓取的未加工文件。**严禁手工修改。**
    *   **`processed/`**: **【活跃】处理后数据池**。已按照城市划分（如 `shenzhen/`, `beijing/` 等）整理归类，同时包含 `trajectories/` 目录存放飞行轨迹。这是目前平台 3D 可视化引擎（Deck.gl）的直接数据源。
    *   **`refs/`**: 参考数据（如有）。
    *   **`Data_Dictionary.md`**: **数据字典**。全域字段名、单位、数据类型的权威说明，请在前端开发时随时查阅本字典中定义的字段。

## 💡 给后续前端与算法开发者的使用提醒
1. 所有的爬虫或下载脚本 (`scripts/`) 现已稳定闭飞，**请不要再重复运行抓取脚本**以免触发公网 API 的封禁。
2. 现有数据的坐标已经被确切地映射到 **WGS84** 经纬度体系上，在计算 3D 碰撞相交时，请直接对接 `buildings_3d.geojson` 的建筑高度 (height) 与无人机的绝对高度 (alt_abs/alt_rel)。
